事情起因本来是这样的，我在网站里加了一张图片。颜色是#EFCAC3，结果网站上呈现出来的时候变色了？？？变成了#A78D88。一脸懵逼，然后我打开控制台看了一下，页面上其实是加了一条rgba(0,0,0,0.3)。那想让显示之后的效果为#EFCAC3就得重新设置原来的颜色了。这就涉及到计算alpha混合。

Alpha混合就是实现一种混合之后的颜色效果。

假设一种不透明东西的颜色是A，另一种透明的东西的颜色是B，那么透过B去看A，看上去的颜色C就是B和A的混合颜色。

可以用下面的这个式子来近似计算，设B物体的透明度为alpha(取值为0-1，0为完全透明，1为完全不透明)

R(C)=alpha∗R(B)+(1−alpha)∗R(A)R(C)=alpha*R(B)+(1-alpha)*R(A)R(C)=alpha∗R(B)+(1−alpha)∗R(A)

G(C)=alpha∗G(B)+(1−alpha)∗G(A)G(C)=alpha*G(B)+(1-alpha)*G(A)G(C)=alpha∗G(B)+(1−alpha)∗G(A)

B(C)=alpha∗B(B)+(1−alpha)∗B(A)B(C)=alpha*B(B)+(1-alpha)*B(A)B(C)=alpha∗B(B)+(1−alpha)∗B(A)

先说明一下，rgba里边的a其实就是alpha值。按百分比来看其取值范围为0-1；但是按数值来看，32位的alpha值的数值的范围是0到255，其中0表示完全透明的颜色，255表示完全不透明的颜色。上边的公式是使用alpha的百分比计算的嗷。

透明度70%：

  • 用百分制表示：alpha = 0.3
  • 用数值表示：alpha = 76.5

我现在想让C的颜色是#EFCAC3也就是rgb(239,202,195)，B的颜色是rgb(0,0,0)，alpha=0.3

R(C)=239=0.3∗0+0.7∗R(A)　　∴R(A)=341R(C)=239=0.3*0+0.7*R(A)　　∴R(A)=341R(C)=239=0.3∗0+0.7∗R(A)　　∴R(A)=341

G(C)=202=0.3∗0+0.7∗G(A)　　∴G(A)=289G(C)=202=0.3*0+0.7*G(A)　　∴G(A)=289G(C)=202=0.3∗0+0.7∗G(A)　　∴G(A)=289

B(C)=195=0.3∗0+0.7∗B(A)　　∴B(A)=279B(C)=195=0.3*0+0.7*B(A)　　∴B(A)=279B(C)=195=0.3∗0+0.7∗B(A)　　∴B(A)=279

且慢，32位的数据最大是255，我算出来的超过255了！那就证明AB两个颜色没办法融合成C。哈哈哈哈哈哈哈哈哈哈哈哈哈白算半天。

没关系，那我重新换个透明蒙板的颜色。
让C的颜色是是rgb(239,202,195)，B的颜色是rgb(255,255,255)，alpha=0.3

R(C)=239=0.3∗255+0.7∗R(A)　　∴R(A)=232R(C)=239=0.3*255+0.7*R(A)　　∴R(A)=232R(C)=239=0.3∗255+0.7∗R(A)　　∴R(A)=232

G(C)=202=0.3∗255+0.7∗G(A)　　∴G(A)=179G(C) = 202 = 0.3 * 255 + 0.7 * G(A) 　　∴ G(A) = 179G(C)=202=0.3∗255+0.7∗G(A)　　∴G(A)=179

B(C)=195=0.3∗255+0.7∗B(A)　　∴B(A)=169B(C)=195=0.3*255+0.7*B(A)　　∴B(A)=169B(C)=195=0.3∗255+0.7∗B(A)　　∴B(A)=169

设置图片颜色为rgb(232,179,167)，透明蒙板的颜色为rgba(255,255,255,0.3)即可达到我想要的效果。
